# 📊 Visualizations - NLP Spam Detection Project

## 📌 Overview

This folder contains all the **graphs and visual outputs** generated during the analysis of the SMS Spam dataset. These visualizations help in understanding data patterns, model performance, and insights derived from NLP techniques.

---

## 📂 Contents

### 🔹 1. Exploratory Data Analysis (EDA)

* **Spam vs Ham Distribution**

  * Shows imbalance in dataset (more ham than spam)
* **Message Length Distribution**

  * Spam messages tend to be longer than ham messages

---

### 🔹 2. Feature Analysis

* **Word Count Distribution**
* **Character Length Comparison**
* **Digit & Special Character Usage**

  * Spam messages often contain numbers and symbols

---

### 🔹 3. POS Tagging Analysis

* Comparison of grammatical structures between spam and ham
* Visual representation of:

  * Nouns
  * Verbs
  * Adjectives
  * Adverbs

---

### 🔹 4. Named Entity Recognition (NER)

* Distribution of entities such as:

  * Organizations
  * Locations
  * Money values
* Helps identify patterns commonly used in spam messages

---

### 🔹 5. N-Gram Analysis

* **Top Bigrams and Trigrams**

  * Examples: "free entry", "call now", "win cash"
* Shows common phrases used in spam vs ham

---

### 🔹 6. Model Performance Visualization

* **Accuracy & F1 Score Comparison Chart**

  * Compares all models side by side
* Helps identify the best performing model

---

### 🔹 7. Confusion Matrices

* Visual representation of model predictions:

  * True Positives (Spam correctly identified)
  * True Negatives (Ham correctly identified)
  * False Positives
  * False Negatives

---

### 🔹 8. Deep Learning Training Graphs

* **Accuracy vs Epochs**
* **Loss vs Epochs**
* Shows model learning behavior and convergence

---

## 📊 Key Insights from Visualizations

* Spam messages are generally longer and more structured
* Frequent use of promotional words like *"free", "win", "offer"*
* Linear models (SVM, Logistic Regression) perform best
* Confusion matrices show low false negatives for best models

---

## 📌 Usage

These visualizations are useful for:

* Understanding dataset characteristics
* Comparing model performance
* Presentations and reports
* Demonstrating NLP concepts visually

---

## 🧠 Conclusion

The visualizations provide strong evidence that proper preprocessing and feature engineering significantly improve model performance in text classification tasks.
